[ 맨땅에 해딩하는 Deno + Oak + React ] jspm.io / esm.sh

얼마전 공부삼아 작업한 내역중 격은 문제를 정리.

Deno 환경에서 Oak Framework를 사용하여 SSR을 사용한 Frontend Server를 작성함.

서버 사이드 렌더링하는 과정에서 시행착오를 겪으면서 개념도 잡고 열심히 만들었음.

테스트 > 페이지 잘 보임.

Deno 환경에서 React를 사용하기 위해 보편적인 가이드와 리포지터리등을 돌면서 정보를 수집.

대표적인 배포경로로써 jspm.io가 안내 됨. 그래서 사용.

1차 이슈. Types 정보가 가이드 되지 않음. 각 import경로 위에 “// deno-types=“으로

각 모듈의 Types정보를 기술한 소스를 연결해 줘야 함.

2차 이슈. Types를 정리한 소스 내의 서브 모듈 임포트 정보가 인식되지 않음. (global 머시기)

각 타입 소스를 다운로드하여 임포트 경로를 수정해서 해결.

테스트 > 페이지 잘 보임.

css 처리를 위해 styled-components를 도입. Types정보도 긁어와서 연결 해줌

3차 이슈. 이놈 웹에 퍼진 소스들과 익스포트가 다르다. Type 에러가 계속 난다.

임포트 된 객체를 로그로 찍어 구조를 보고 임포트.

테스트 > 페이지 잘 보임. 스타일도 잘 먹음.

계속 타입정보를 긁어와서 연결해 줘야 하나? 귀차니즘에 검색을 더해봄.

기타 몇몇 프로젝트에서 esm.sh를 사용 임포트 시킴. 그래서 변경

각 모듈별로 Types정보를 연결 안해줘도 가이드가 잘됨. 아싸 좋구나

테스트 > 잘보이나 스타일이 안 먹음.

4차 이슈. 왜 안먹지? 봤더니 Deno의 API가 딸려 들어감. Web인데…

제외 방법을 찾아봄. 못 찾음. 실패. 그래서 jspm으로 롤백 후 중단. 마음의 평화가 필요함.
